插件采用 poi-tl（poi template language） Word 模板引擎进行渲染，当前使用版本为：1.12.2。
详细模板语法请参考：poi-tl 模板引擎
  1. 可用基础变量：
    1. {{title}} 文档标题
    2. {{author}} 作者
    3. {{date}} 当前日期
    4. {{version}} 版本号
    5. {{toc}} 目录占位符
  2. 可用集合变量：
    1. {{dataSources}} 所有选中数据源（含库和表）：
      1. das 数据源原始对象
      2. name 数据源名称
      3. comment 数据源备注
      4. dialect 数据源方言
      5. version 数据源版本
      6. namespaces 命名空间集合（关系库中通常为 schema）：
        1. das 命名空间原始对象
        2. name 命名空间名称
        3. comment 命名空间注释
        4. tables 数据对象集合（物理表/视图/物化视图）：
          1. das 数据对象原始对象
          2. name 名称
          3. comment 注释
          4. objectKind 对象类型（TABLE/VIEW/MATERIALIZED_VIEW）
          5. schema 所属 schema
          6. primaryKey 主键
          7. columns 列集合（name/comment/type/defaultValue/isPrimaryKey 等）
    2. 若视图/物化视图列注释为空，生成前会尝试使用同名物理表列注释回填。
    3. {{ds4table}} 与 {{dataSources}} 等价（便于 LoopRowTableRenderPolicy）。
    4. {{ns4table}} 与 namespaces 等价（便于 LoopRowTableRenderPolicy）。
    5. 分章节变量（按 objectKind）：
      1. Tables：{{dataSourcesTables}} / {{namespacesTables}} / {{ds4tableTables}} / {{ns4tableTables}}
      2. Views：{{dataSourcesViews}} / {{namespacesViews}} / {{ds4tableViews}} / {{ns4tableViews}}
      3. Materialized Views：{{dataSourcesMaterializedViews}} / {{namespacesMaterializedViews}} / {{ds4tableMaterializedViews}} / {{ns4tableMaterializedViews}}
    6. 数据量较大时会分片并发渲染后自动合并文档，并清理临时目录。
  3. 排序功能说明（可选）：
    1. 点击主窗口左下角 排序配置 可展开左侧配置面板。
    2. 排序来源支持：
      1. 不启用排序：保持原有顺序（默认）。
      2. 多行输入：每行一个表名，支持空行与 # 注释。
      3. TXT 文件：读取 UTF-8 文本，每行一个表名。
      4. 列表拖动：列表内展示全部表名，初始顺序为在元数据中的首次出现顺序，与多行/TXT 一样对表名去重。可在列表中多选，将选中的多行整组拖放到目标位置以调整顺序；行首 序号. 仅为界面显示，实际参与顺序的仍是表名字符串。
    3. 排序规则：命中表按给定顺序排序；未命中表自动排在后面。
    4. 排序大小写不敏感，重复表名自动去重（保留首次出现）。
    5. 若不同 schema 下存在同名表，排序以表名为键，与多行/TXT/列表一致，请注意与 schema 的对应关系是否满足你的预期。
  4. 结构编排变量速查（poi-tl）：
    1. 核心新增变量：
      1. {{plan}}：完整编排树（根节点列表在 plan.roots）。
      2. {{vars}}：兼容变量映射；父节点变量值为 List<DocNode>，叶子变量值为 DocNode。
      3. {{varNodes}}：统一节点映射，父/叶均返回 DocNode。
      4. {{varItems}}：统一集合映射，父/叶均返回 List<DocNode>（叶子为单元素列表）。
    2. 推荐优先使用 {{varItems.xxx}} 做循环渲染（父/叶语义一致）。
    3. 推荐使用 {{varNodes.xxx}} 读取节点元信息（显示名称、类型、变量名等）。
    4. 常用节点字段：
      1. displayName 显示名称
      2. type 节点类型（GROUP / DATABASE / NAMESPACE / TABLE / VIEW / MAT_VIEW）
      3. variableName 变量名
      4. order 同级排序号
      5. enabled 是否启用
      6. ref.dataSource / ref.namespace / ref.objectName / ref.objectKind 对象来源信息
    5. 语义规则：
      1. 节点被禁用（enabled=false）后，不会进入 vars / varNodes / varItems。
      2. 父节点变量在 varItems 中表示其“所有启用后代叶子节点”的聚合结果。
      3. 叶子节点变量在 varItems 中为单元素列表，便于统一模板写法。
    6. 示例（可直接参考）：
      1. 分组标题：{{varNodes.core_group.displayName}}
      2. 分组循环： {{#varItems.core_group}} ... {{displayName}} / {{type}} ... {{/varItems.core_group}}
      3. 单表统一写法： {{#varItems.table_user}} ... {{ref.namespace}} ... {{/varItems.table_user}}
      4. 整棵树遍历： {{#plan.roots}} ... {{displayName}} ... {{#children}} ... {{/children}} {{/plan.roots}}
