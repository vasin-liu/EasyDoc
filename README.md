# EasyDoc（数据库文档生成插件）

基于 IntelliJ IDEA 的数据库文档生成插件，使用 **poi-tl** 将选中库表元数据渲染为 **Word（.docx）**，支持内置/自定义模板。

## 运行与依赖

| 项目 | 说明 |
| --- | --- |
| Gradle | 8.6 |
| Kotlin | 1.9.22 |
| Gradle IntelliJ 插件 | 1.17.2 |
| JDK 构建 | 17（见 `build.gradle.kts` / IDE 项目设置） |
| poi-tl | 1.12.2 |

## IDE 与插件依赖

- **最低平台版本**：`232` 对应 **2023.2.x 及以上**（与 `build.gradle.kts` 中 `sinceBuild` 一致时为准）。
- **必需插件/模块**：`Database`（`com.intellij.database`）、Java 平台等（见 `META-INF/plugin.xml` 的 `<depends>`）。

## 从何处打开

在 **Database** 工具窗口中，选中一个或多个 **表、视图/物化视图、或 Schema（命名空间）**，在弹出菜单中执行 **EasyDoc**（`DatabaseViewPopupMenu`），打开生成对话框。仅当选中项可解析为数据库对象时，菜单项会可用。

## 主界面功能

1. **语言**：中文 / English，影响界面与内置帮助（`?`）文案。
2. **模板**  
   - **默认**：使用插件内置的 `.docx` 模板。  
   - **自定义**：选择本机 Word 模板文件（需按 poi-tl 语法与插件暴露的数据模型编写，详见内置帮助 **模板说明**）。
3. **数据库**：以表格形式展示当前选区汇总后的**数据源 / 库（命名空间）**；可编辑「数据库描述」等字段，会参与文档变量。
4. **文档信息**：文档标题、作者、版本号；保存路径为目录，生成文件名为 `<标题>.docx`。
5. **排序（可选）**  
   点击左下角 **「排序配置」** 可展开/收起左侧排序面板。生成前若启用非「不启用排序」，会按你指定顺序调整数据模型中的表/命名空间展示顺序。  
   - **不启用排序**：使用 IDE/插件解析后的原始顺序。  
   - **多行输入**：每行一个**表名**；空行与以 `#` 开头的行忽略；**大小写不敏感**；**重复表名去重，保留先出现的顺序**。  
   - **TXT 文件**：UTF-8 文本，每行一个表名，规则同多行输入。  
   - **列表拖动**：列表初始为**全部表名**在元数据中的**首次出现顺序**（同名表去重，规则与上两项一致）；**支持多选**，将选中行**整组拖放**到目标插入位置以调整顺序；行首 **`序号.` 仅为界面展示**，落盘/生成时仍只按**表名**参与排序。  
   **规则补充**：在排序列表/文件中**被明确写出**的表按该顺序；未写出的表排在后面；**命名空间/Schema 间的顺序**会受其中表的排序结果间接影响。若不同 schema 下存在**同名表**，排序键为表名本身（与现有多行/文件行为一致），请注意歧义。  
6. 内置帮助（对话框 **?** 按钮）说明 **poi-tl 版本、模板变量、集合与分表变量、分片合并**等，与下节「帮助文档」为同一套内容，随界面语言切换 `help_zh_CN.html` / `help_en.html`。

## 其他行为（摘要）

- 视图/物化视图在列上缺少注释时，会尽量从**同库同名物理表**列上继承列注释。  
- 元数据与文档**体量较大**时，可能采用分片生成再合并、并清理临时目录（详见代码与帮助说明）。

## 帮助文档

- **资源位置**：`src/main/resources/help_zh_CN.html`、`src/main/resources/help_en.html`  
- **在 IDE 中打开**：主对话框中点击 **?**，由 `HelpDialog` 按当前语言加载上面对应文件。  
  主要内容包括：poi-tl 与官方文档链接、**基础变量/集合与 dataSources 结构**、按表/视图/物化视图拆分的**章节变量**、**排序**各项说明、以及大数据量时合并行为说明。

## 参与构建

```text
# Windows 示例
gradlew buildPlugin
```

因环境差异（如 Gradle、JDK、内网仓库），以本机能成功拉取 `gradle-intellij-plugin` 等依赖为准。沙箱/调试目录可参见 `build.gradle.kts` 中 `intellij { sandboxDir ... }`（默认指向仓库内 `idea-sandbox` 等，勿将纯本地路径写入对外文档的强制要求，此处仅作开发提示）。
